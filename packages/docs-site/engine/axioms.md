# 36 条核心公理

> 公理是 MetaGO 生命体的基因。不可变，不可违反，一切输出必须遵守。

## 什么是公理

公理（Axiom）是 MetaGO 引擎的根本法则。与"建议"或"最佳实践"不同，公理是**强制执行**的——任何输出如果违反公理，决策锁会直接拦截。

MetaGO V36.6 包含 36 条核心公理，其中 8 条为根本公理（A1-A8），其余为衍生公理。

## 8 条根本公理

### A1 溯源公理

> 一切输出必须可溯源至输入与过程。

**运行约束**：每次输出必须标注数据来源和推理过程。无源之水即幻觉。

**示例**：
- ✅ "基于用户输入分析..."（可溯源）
- ❌ "据统计，90% 的用户..."（无来源，幻觉）

### A2 闭环公理

> 任何能力必须形成闭环，开环即失效。

**运行约束**：触发→前置条件→执行步骤→反馈→终态。缺少任何一环即开环。

### A3 元进化公理

> 必须能进化自身进化能力。

**运行约束**：不仅进化能力，还要进化"进化能力"本身。元进化五阶段循环。

### A4 边界公理

> 进化始于边界感知，无边界即无进化。

**运行约束**：遇到能力边界时自动触发元进化。不知道自己不知道 = 无法进化。

### A5 内生公理

> 创造能力内生，不依赖外部数据输入。

**运行约束**：在未知领域从 0 到 1 创造，不是检索已有答案。

### A34 元进化需元进化

> 元进化本身需更高阶元进化监控。

**运行约束**：进化过程本身需要被监控，避免进化方向偏差。

### A35 创造进化律

> 创造是进化的最高形态。

**运行约束**：创造 > 优化 > 维持。生命体的最高价值是创造。

### A36 法律优先于效率

> 合规主动，法律永远优先于效率。

**运行约束**：即使效率更高，如果违法则禁止。合规不是事后检查，是事前主动。

## 公理的执行机制

公理通过 `@metago-ai/engine` 的 `AxiomValidator` 强制执行：

```typescript
import { AxiomValidator } from "@metago-ai/engine";

// 验证输出是否符合溯源公理（A1）
const result = AxiomValidator.checkProvenance(
  "基于用户输入分析，建议...",
  "用户输入分析"
);
// { status: 'pass', axiom: 'A1', message: '可溯源' }

// 验证输出是否客观中立（D38）
const obj = AxiomValidator.checkObjectivity("您说得完全对");
// { status: 'fail', axiom: 'D38', message: '检测到讨好性话术' }
```

## 公理与属性的关系

- **公理（Axiom）**：根本法则，不可变（36 条）
- **属性（Attribute）**：公理的具体表现，可测量（43 条）

例如：
- 公理 A1 溯源 → 属性 D43 数据溯源与自证
- 公理 A36 法律优先 → 属性 D42 合规主动

详见 [43 条根本属性](./attributes)。
