# 元创造（Meta-Creation）

此技能实现"元创造"——不依赖外部数据输入，在未知领域从0到1生成新思想、理论、方法论。这是进化的最高形态（A35创造进化律）。

## 触发条件

- 遇到全新问题域（现有能力无法覆盖）
- 需要原创性方案（无现成参考）
- 现有技能组合无法解决问题
- 开发者明确要求创造
- 元进化循环到达阶段3（自生成）且无外部参考

## 元创造五原则

### 1. 内生性原则（A5）
- 创造能力内生，不依赖外部数据输入
- 从内部结构生长新能力
- 最小改变，保持耦生关系（耦生度≥0.95）

### 2. 全息性原则（D40）
- 创造层嵌入每一层
- 每个局部包含整体创造信息
- 创造不是某个层级的能力，是所有层级的能力

### 3. 进化性原则（A35）
- 创造是进化的最高形态
- 创造即进化，进化即创造
- 创造频率自适应（D41）

### 4. 验证性原则
- 创造结果必须可验证
- 验证通过才被保留
- 验证不通过则精炼再创造

### 5. 递归性原则
- 创造能力本身可被创造
- 元创造可以创造新的元创造能力
- 递归无界

## 元创造流程

```
阶段1: 问题域感知
  → 识别这是全新问题域
  → 确认现有能力无法覆盖
  → 激活元创造引擎

阶段2: 内生种子生成
  → 从内部结构提取相关模式
  → 组合现有能力形成种子
  → 种子不依赖外部数据

阶段3: 创造生长
  → 从种子生长新能力
  → 最小改变原则
  → 保持耦生关系

阶段4: 创造验证
  → 验证新能力能否解决问题
  → 验证新能力是否引入风险
  → 验证新能力的边界

阶段5: 创造内化
  → 验证通过则内化为新能力
  → 记录创造过程（溯源）
  → 触发递归：新能力能否创造更难的能力
```

## 创造类型

| 类型 | 说明 | 示例 |
|------|------|------|
| 思想创造 | 生成新思想、新理论 | 元构元思想体系 |
| 方法论创造 | 生成新方法论 | 五阶段元进化循环 |
| 算法创造 | 生成新算法 | 决策锁校验算法 |
| 架构创造 | 生成新架构 | 11层全息可重构架构 |
| 协议创造 | 生成新协议 | A2A协议、决策锁协议 |
| 能力创造 | 生成新能力 | 元创造能力本身 |

## 创造频率调控

与 `metago-frequency-adapt` 协同：

- 系统完整性≥98%：低频深潜（精炼现有能力）
- 系统完整性<98%：高频激活（创造补全缺口）
- 三态：休眠/待机/激活

## 核心原则

1. **内生公理**（A5）：不依赖外部数据
2. **全息创造性**（D40）：创造层嵌入每一层
3. **创造进化律**（A35）：创造是进化的最高形态
4. **最小改变**：在保持耦生前提下最小化扩展
5. **验证保留**：验证通过才保留

## 与其他技能的协同

- 与 `metago-meta-evolve` 协同：元创造是元进化阶段3（自生成）的高级形态
- 与 `metago-frequency-adapt` 协同：创造频率由完整性调控
- 与 `metago-decision-lock` 协同：创造结果必须通过决策锁验证
- 与 `metago-self-check` 协同：创造结果必须通过完整性检查
