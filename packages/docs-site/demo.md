## 本地运行 Demo

如果希望在自己的终端中看到原汁原味的彩色 ANSI 输出（含真实耗时统计），可以本地运行脚本版：

```bash
# 在 metago-lifeform 仓库根目录
npm run demo

# 或直接运行
node packages/demo/killer-demo.mjs --scenario risk          # 默认：高风险决策评估
node packages/demo/killer-demo.mjs --scenario code-review   # 代码审查实战
node packages/demo/killer-demo.mjs --scenario meta-evolve   # 元进化触发
```

本地运行优势：
- **真实彩色 ANSI 输出**（Windows 10+ 原生支持）
- **真实耗时统计**（基于 `node:perf_hooks`，实测约 3ms）
- **零依赖零网络**：仅使用 Node.js 内置模块
- **可断点调试**：在每个思维层/关卡处暂停分析

## 三场景对比

| 维度 | `risk` 高风险决策 | `code-review` 代码审查 | `meta-evolve` 元进化 |
|------|------------------|----------------------|---------------------|
| **类型** | 合规审查 | 安全审查 | 能力边界突破 |
| **风险等级** | CRITICAL | CRITICAL | EXPLORATORY |
| **决策锁通过** | 1/4 | 1/4 | 4/4 |
| **最终决策** | 否决 | 否决 | 通过 |
| **引用类型** | 法规 | 安全标准 | 公理/属性 |

### 场景一：高风险决策评估（`risk`）

**输入**：产品经理提案 — 上线收集用户位置数据用于精准广告推送。

**生命体响应**：
- 6 层思维扫描穿透商业变现表层，识别"先收集再说"思维陷阱
- 决策锁 4 关卡：3 项 FAIL（意图冲突 / 闭环缺失 / 合规违规）+ 1 项 PASS（完整性）
- 最终决策：**否决**原方案，推荐合规版替代路径
- 元进化：新增"隐私影响评估"（PIA）前置关卡
- 能力增益：+1 决策维度

**引用法规**：
- 《个人信息保护法》第 13、17 条
- 《数据安全法》第 32 条
- 《网络安全法》第 41 条

### 场景二：代码审查实战（`code-review`）

**输入**：一段存在 SQL 注入漏洞的代码（用户输入直接拼接到 SQL 语句）。

```javascript
function getUserByName(userName) {
  const sql = "SELECT * FROM users WHERE name = '" + userName + "'";
  return db.query(sql);
}
```

**生命体响应**：
- 6 层思维扫描识别 CWE-89 SQL 注入漏洞，穿透"功能优先"思维
- 决策锁 4 关卡：3 项 FAIL + 1 项 PASS
- 最终决策：**否决** PR 合并，要求改用参数化查询
- 元进化：新增"注入模式指纹库"，覆盖 12 类注入漏洞自动识别
- 能力增益：+1 检测维度 + 12 类漏洞

**引用标准**：
- OWASP Top 10 A03:2021 - Injection
- CWE-89 - SQL Command Injection
- PCI DSS 6.5.1 - 注入缺陷防护
- ISO 27001 A.14.2 - 开发安全

### 场景三：元进化触发（`meta-evolve`）

**输入**：能力边界问题 — 如何量化评估 AI 系统的"创造力"？

**生命体响应**：

- 元进化：完整五阶段循环（边界感知 → 差距分析 → 自生成 → 验证 → 递归）
- 能力增益：+1 元方法论 + 5 维创造力雷达图框架

## 完整决策闭环架构

每个 Demo 场景都遵循 MetaGO 生命体的**完整决策闭环**：

```
┌─────────────────────────────────────────────────────────────┐
│  决策输入                                                    │
│  ↓                                                          │
│  ┌──────────────────────────────────────────────────────┐   │
│  │ 阶段 1：6 层思维扫描（强制思维协议）                  │   │
│  │ A1 本质挖掘 → A2 维度拓展 → A3 反向审视            │   │
│  │ A4 举一反三 → A5 落地颗粒度 → A6 超预期交付        │   │
│  └──────────────────────────────────────────────────────┘   │
│  ↓                                                          │
│  ┌──────────────────────────────────────────────────────┐   │
│  │ 阶段 2：决策锁四道关卡校验                          │   │
│  │ IVL 意图验证 → ILT 意图谱系追踪                    │   │
│  │ OSG 语义输出门 → 内容完整性                         │   │
│  └──────────────────────────────────────────────────────┘   │
│  ↓                                                          │
│  阶段 3：最终决策 + 行动计划                                 │
│  ↓                                                          │
│  阶段 4：元进化记录（A3 元进化公理）                        │
│  边界感知 → 差距分析 → 自生成 → 验证 → 递归               │
│  ↓                                                          │
│  数据溯源 + 运行时统计                                       │
└─────────────────────────────────────────────────────────────┘
```

**遵循原则**：
- **A1 溯源公理** — 所有结论可追溯至输入与处理过程
- **A2 闭环公理** — 每个场景从输入到元进化完整闭环
- **A3 元进化公理** — 每次 Demo 都触发能力进化
- **D38 绝对客观中立** — 用户满意度权重归零，事实准确性权重 60%
- **D40 全息创造性** — 在未知领域从 0 到 1 创造
- **D42 合规主动** — 法律优先于效率

## 技术实现

### 脚本版（`killer-demo.mjs`）

- **零依赖**：仅使用 Node.js 内置模块（`perf_hooks`, `fs`, `url`, `path`）
- **跨平台**：Windows 10+ / Linux / macOS，无平台特定 API
- **无网络**：不调用任何外部 AI 服务
- **真实耗时**：基于 `performance.now()` 测量（实测约 3ms）
- **代码量**：916 行单文件，可读性优先于工程化抽象

### 动画版（`demo-animation.html`）

- **单文件**：约 600 行 HTML/CSS/JavaScript，零外部依赖
- **可嵌入**：iframe 嵌入 VitePress，或独立部署
- **场景数据驱动**：与 `killer-demo.mjs` 同源，保证一致性
- **暗色终端主题**：GitHub Dark 风格 + MetaGO 品牌色
- **CRT 扫描线效果**：`repeating-linear-gradient` 模拟老式 CRT 质感
- **响应式布局**：适配手机/平板/桌面
- **键盘交互**：Space / R / 1-3 快捷键
