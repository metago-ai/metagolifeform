# AI Agent 运行时验证强制规范（模板）

> 把此文件的内容复制到你的 AI Agent 系统提示词中，强制 AI 不可绕过运行时验证。

## 核心铁律

**编译通过 ≠ 运行通过。tsc 0 错误 + build 成功只是技术层的必要条件，不是充分条件。**

## 三层验证架构

| 层级 | 内容 | 工具 |
|------|------|------|
| L1 技术层 | 类型检查、构建、依赖 | `tsc -b` + `vite build` |
| L2 业务层 | 核心功能端到端可用 | `npm run verify` |
| L3 链路层 | 跨页面/跨功能关联完整 | HTTP HEAD |

**三层全部通过 = 任务完成。任意一层未通过 = 任务未完成。**

## 五问自检（输出"任务完成"前必答）

1. 我运行了 `npm run verify` 吗？
2. verify 有 FAIL 吗？
3. 报告含验证小节吗？
4. 每项 ✅ 都有证据吗？
5. 我做业务层验证了吗？

**任何一问答"否" = 禁止宣告完成。**

## 反绕过条款

以下行为视为绕过：
- "应该没问题" / "逻辑上正确" / "之前验证过" / "理论上没问题"
- 只做 tsc + build，不做业务层验证
- 清单全勾 ✅ 但无执行证据
- 发现问题但隐瞒不报

## 强制执行回路

```
接收任务 → 执行 → tsc（硬门1）→ build（硬门2）→ verify（硬门3）→ 部署 → HTTP 验证（硬门4）→ 报告 → 5 问自检 → 宣告完成
```

跳过任何硬门 = 任务未完成。
