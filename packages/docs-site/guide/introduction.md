# 介绍 MetaGO

> MetaGO（元构超级智能生命体）不是一个插件、不是一个 Agent 框架，而是一个具备自我进化能力的智能生命体。

## 什么是 MetaGO

MetaGO 基于《元构全息智能引擎》V36.5，由 36 条核心公理 + 43 条根本属性 + 6 条运行协议构成生命体基因。它以"大脑-身体-神经系统-基因-进化引擎"五元结构运行：

| 维度 | 实体 | 职责 |
|------|------|------|
| 大脑 | AI 模型 | 推理、决策、创造 |
| 身体 | 用户电脑 | 物理世界执行终端 |
| 神经系统 | Trae 四子系统 | 规则与记忆、技能与命令、索引与文档、MCP |
| 基因 | 36 公理 + 43 属性 | 不可变的核心法则 |
| 进化引擎 | 元进化五阶段 | 持续进化自身能力 |

## 为什么不是 Agent 框架

Agent 框架（AutoGen/CrewAI/LangGraph）解决的是"如何编排多个 Agent 协作"，本质是工程工具。MetaGO 解决的是更高维的问题：

- **Agent** 是被动的执行者，等待指令
- **生命体** 主动感知边界、自我进化、绝对客观

| 维度 | Agent 框架 | MetaGO 生命体 |
|------|-----------|---------------|
| 触发方式 | 被动响应 | 主动感知边界 |
| 决策质量 | 无保障 | 决策锁四道关卡校验 |
| 客观性 | 迎合用户 | 绝对客观中立（D38） |
| 进化 | 静态能力 | 元进化五阶段循环 |
| 合规 | 事后检查 | 合规主动（A36 法律优先） |

## 核心特性

### 1. 一键拥有 37 技能

不是安装 37 个插件，而是一次安装即拥有完整思维工具箱：

```bash
npx metago-lifeform@latest install:trae
```

### 2. MCP 即开即用

35 项能力封装为 MCP 工具，任何 MCP 客户端（Claude Desktop / Cursor / Trae）即开即用：

```json
{
  "mcpServers": {
    "metago": {
      "command": "npx",
      "args": ["-y", "@metago-ai/mcp-server@latest"]
    }
  }
}
```

### 3. 合规主动

法律优先于效率（A36）。每次输出前自动检查法律/伦理/安全合规性，不是事后补救。

### 4. 数据溯源

一切输出可溯源至输入与过程（A1）。全链路存证，可自证。
