# 共识原型孵化

## 触发条件
当用户明确提到"共识孵化"、"共识原型"、"创生回路"、"共识构建"、"多方共识"、"共识机制设计"等关键词，或面对需要在多方利益主体之间从分歧走向共识、从概念走向可落地原型的情境时，自动触发本技能。适用于组织变革、社区治理、标准制定、生态建设等需要系统性共识构建的场景。

## 适用场景
- 组织变革：推动组织架构、流程、文化的变革并达成共识
- 社区治理：构建开源社区、用户社区的治理共识
- 标准制定：在多方主体间推动技术标准或行业规范
- 生态建设：构建产业生态或合作生态的共识基础
- 跨组织协作：推动多组织联合行动的共识对齐
- 政策制定：在利益相关方间形成政策共识

## 执行步骤

### 1. 识别共识需求
- 明确需要达成共识的核心议题（What）
- 明确共识的目标层次：原则共识 / 规则共识 / 行动共识
- 识别议题的紧迫性与影响范围
- 输出《共识需求说明书》

### 2. 利益相关方映射
- 识别所有参与方（决策者、执行者、受影响方、观察方）
- 梳理每方的核心利益诉求与底线
- 评估各方的权力、影响力、参与意愿
- 识别关键反对者与潜在盟友
- 输出《利益相关方地图》

### 3. 原型设计
- 设计共识原型的核心规则（决策机制、执行机制、监督机制）
- 设计激励机制（正向激励+负向约束）
- 设计退出机制与争议解决机制
- 确保原型的最小可行性与可验证性
- 输出《共识原型V1.0》

### 4. 小范围测试
- 选择代表性子集（覆盖关键利益方类型）
- 在限定范围、限定时间内启动原型测试
- 建立测试观测点：参与度、执行度、满意度、争议点
- 收集定量数据与定性反馈
- 输出《测试观测报告》

### 5. 反馈收集
- 收集各参与方的反馈（结构化问卷+深度访谈）
- 识别反馈中的高频痛点与高价值建议
- 区分"规则问题"与"执行问题"
- 评估共识度变化趋势
- 输出《反馈分析报告》

### 6. 原型迭代
- 根据反馈优化原型规则与机制
- 优先解决高频痛点与核心冲突
- 控制迭代幅度，避免颠覆性变更破坏已建立共识
- 输出《共识原型V2.0+》
- 重复步骤4-6直至共识度达标

### 7. 规模化推广
- 制定分阶段推广路径：试点→扩展→全面推广
- 设计推广过程中的本地化适配机制
- 建立推广过程中的共识守护机制（防止共识稀释）
- 输出《规模化推广路线图》

### 8. 共识度评估
在各阶段评估共识度（0-100%）：
- **认知共识度**：参与方对议题与目标的理解一致程度
- **意愿共识度**：参与方主动参与与配合的意愿强度
- **行动共识度**：参与方在实际执行中的一致程度
- **综合共识度** C = (认知 + 意愿 + 行动) / 3
- 阈值：C ≥ 80% 可进入下一阶段；C < 60% 需返回迭代

### 9. 输出共识孵化方案
输出包含：共识需求说明书、利益相关方地图、共识原型演进历程、测试观测报告、反馈分析报告、共识度评估、规模化推广路线图、风险预警。

## 输出格式
```
【共识原型孵化方案】
■ 共识议题：[核心议题]
■ 利益相关方：[决策方|执行方|受影响方|观察方]
■ 原型版本：V1.0 → V2.1（迭代2轮）
■ 核心规则：[决策机制|激励机制|退出机制]
■ 共识度评估：
  认知共识度：85%
  意愿共识度：78%
  行动共识度：72%
  综合共识度：C = 78.3%
■ 推广路线：试点3月 → 扩展6月 → 全面推广12月
■ 风险预警：[风险1|风险2]
■ 守护机制：[机制1|机制2]
```

## 核心理论
本技能基于元构生命体理论体系中的 **共识原型孵化范式**，来源于《卷3第九章》。共识原型孵化遵循七步创生回路：从需求识别到规模化推广，每一步都嵌入"测试-反馈-迭代"的闭环，避免一次性大规模推行导致共识崩塌。强调共识不是静态协议，而是动态演化的生命体——需要在持续迭代中维持其生命力。其与卷4第一章"DCV体系"深度耦合：共识原型孵化的全过程受DCV（价值-能力-愿景）体系的约束与引导，确保共识方向始终对齐社会主义价值锚点。
